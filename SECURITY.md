# Security Policy

## Supported Versions

The `main` branch receives security fixes. Older tags are not supported.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security reports.

Instead, email the maintainers with:

- A description of the issue and its impact.
- Steps to reproduce or proof-of-concept code.
- Affected commit or release version.

You can expect an acknowledgment within 72 hours and a status update within
seven days while we investigate and prepare a fix.

## Disclosure

We coordinate disclosure with the reporter. Once a patch is released, we
publish an advisory on this repository's Security tab.
